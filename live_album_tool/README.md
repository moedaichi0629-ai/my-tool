
# ライブ思い出アルバム 自動生成ツール

ライブ参加時に撮影した写真と基本情報を入力するだけで、  
Googleドキュメント上にライブ思い出アルバムを自動生成・保存するツールです。

---

## 1. このツールでできること

- ターミナルにライブ名・参加日・会場・写真フォルダのパスを入力するだけで自動処理
- 写真フォルダ内の `.jpg` / `.jpeg` / `.png` ファイルを自動収集（5〜20枚程度を想定）
- Google Driveに写真をアップロードし、Googleドキュメントにアルバムとして保存
- 完成したドキュメントのURLをターミナルに表示

**Googleドキュメントに出力される内容:**

```
ライブ思い出アルバム  ← 大見出し

ライブ名　：○○バンド LIVE TOUR 2024
参加日　　：2024年6月1日
会場　　　：東京ドーム

写真一覧  ← 中見出し
[写真1]
[写真2]
 ...
```

---

## 2. 必要なライブラリ

以下のコマンドでまとめてインストールできます。

```bash
pip install -r requirements.txt
```

インストールされるライブラリ：

| ライブラリ | 用途 |
|---|---|
| google-auth | Google APIの認証処理 |
| google-auth-oauthlib | OAuthログインフロー |
| google-auth-httplib2 | HTTPリクエスト処理 |
| google-api-python-client | Docs・Drive API操作 |

## 3. Google Cloudで必要な設定

このツールを使うには、Google Cloud Console での設定が必要です。  
以下の手順を**一度だけ**行ってください。

### 手順1: Google Cloud Consoleにアクセス

1. ブラウザで [https://console.cloud.google.com/](https://console.cloud.google.com/) を開く
2. Googleアカウントでログインする

### 手順2: プロジェクトを作成する

1. 画面上部の「プロジェクトを選択」をクリック
2. 右上の「新しいプロジェクト」をクリック
3. プロジェクト名（例: `live-album-tool`）を入力して「作成」をクリック
4. 作成したプロジェクトを選択する

### 手順3: APIを有効化する

**Google Docs API の有効化:**
1. 左メニューの「APIとサービス」→「ライブラリ」をクリック
2. 検索欄に「Google Docs API」と入力
3. 「Google Docs API」をクリック→「有効にする」をクリック

**Google Drive API の有効化:**
1. 同じ手順で「Google Drive API」を検索
2. 「Google Drive API」をクリック→「有効にする」をクリック

### 手順4: OAuth同意画面を設定する

1. 左メニューの「APIとサービス」→「OAuth同意画面」をクリック
2. 「外部」を選択して「作成」をクリック
3. 以下を入力して「保存して次へ」をクリック:
   - アプリ名: `ライブアルバムツール`（任意）
   - ユーザーサポートメール: 自分のGoogleアドレス
   - デベロッパーの連絡先情報: 自分のGoogleアドレス
4. 「スコープ」の画面は何も変更せず「保存して次へ」をクリック
5. 「テストユーザー」の画面で「+ ADD USERS」をクリックし、  
   自分のGoogleアカウントのメールアドレスを追加して「保存して次へ」をクリック
6. 最後に「ダッシュボードに戻る」をクリック

### 手順5: OAuthクライアントIDを作成する

1. 左メニューの「APIとサービス」→「認証情報」をクリック
2. 上部の「＋ 認証情報を作成」→「OAuthクライアントID」をクリック
3. アプリケーションの種類: 「デスクトップアプリ」を選択
4. 名前は任意（例: `live-album`）
5. 「作成」をクリック
6. 表示されたウィンドウの「JSONをダウンロード」をクリック

## 4. credentials.jsonの置き場所

ダウンロードしたJSONファイルを以下の手順で配置してください。

1. ダウンロードしたファイルの名前を `credentials.json` に変更する
2. 以下のフォルダに保存する:

```
live_album_tool/
├─ main.py
├─ requirements.txt
├─ README.md
├─ credentials.json   ← ここに置く（Google Cloudからダウンロード）
├─ token.pickle       ← 初回認証後に自動生成されます（削除しないこと）
└─ images/            ← 写真サンプルを置けます（任意）
```

## 6. エラーが出たときの確認ポイント

### `credentials.json が見つかりません` と表示される

→ `credentials.json` が `main.py` と**同じフォルダ**にあるか確認してください。  
→ ファイル名が `credentials.json` になっているか確認してください（ダウンロード直後は別の名前の場合があります）。

### `フォルダが見つかりません` と表示される

→ 写真フォルダのパスが正しいか確認してください。  
→ Windowsの場合、エクスプローラーでフォルダを開き、アドレスバーのパスをコピー&ペーストするのがおすすめです。

### ブラウザのログイン後に `アクセスできません` と表示される

→ OAuth同意画面の「テストユーザー」に自分のGoogleアカウントが  
　登録されているか確認してください（手順4のステップ5）。

### `google.auth.exceptions.RefreshError` エラーが出る

→ `token.pickle` を削除してから再実行してください。  
　ブラウザが開き、再度ログインできます。

```bash
# Windows の場合
del token.pickle
```

### `HttpError 403` が出る（権限エラー）

→ Google Docs API と Google Drive API の両方が有効になっているか確認してください（手順3）。

### 画像がドキュメントに表示されない

→ Drive にアップロードされた画像の共有設定が「リンクを知っている全員が閲覧可能」  
　になっているか、Google Drive を開いて確認してください。  
→ ドキュメントを開き直すと表示される場合があります（反映に数秒かかることがあります）。

## 7. 今後追加できる機能

コードを整理して拡張しやすくしてあります。以下の機能を追加できます。

| 機能 | ヒント |
|---|---|
| 感想・メモの記録 | `get_user_input()` に `memo` フィールドを追加し、`add_info_text()` で書き込む |
| セットリストの記録 | 入力を複数行テキストとして受け取り、箇条書きで挿入する |
| 写真のキャプション入力 | `get_images_from_folder()` でファイル名と一緒にコメントを入力できるように変更 |
| 画像サイズの変更 | `main.py` の `IMAGE_WIDTH_PT` / `IMAGE_HEIGHT_PT` の値を変更する |
| 特定フォルダへの保存 | Drive API の `parents` フィールドでフォルダIDを指定する |
| 複数ライブの一括処理 | `main()` をループ処理に変更し、繰り返し実行できるようにする |
| ドキュメントのデザイン変更 | `updateTextStyle` リクエストでフォント・色・サイズを指定する |
