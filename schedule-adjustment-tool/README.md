# 📅 日程調整支援ツール

Googleカレンダーと連携し、空き時間の候補日と送信用文章を自動生成するWebアプリです。

---

## どんな課題を解決するか

フリーランス・経営者・営業の方は、商談・面談・打ち合わせのたびに日程調整が発生します。

| 課題 | このツールの解決方法 |
|------|---------------------|
| 空いている時間を探すのが面倒 | Googleカレンダーを自動で確認 |
| 候補日を複数出すのに時間がかかる | 空き時間を最大5件自動提案 |
| 相手に送る文章を毎回考える | LINE・メール用の文章を自動生成 |
| 予定確定後のカレンダー登録が手間 | アプリから直接カレンダーに登録 |
| ダブルブッキングが心配 | カレンダーを確認してから候補を出す |

---

## 主な機能

1. **Googleログイン** — Google OAuth 2.0でセキュアに認証
2. **空き時間の抽出** — 希望期間・時間帯・打ち合わせ時間を指定して空き時間を検索
3. **候補日の提案** — 空き時間を最大5件リストアップ
4. **送信文章の生成** — LINE向け・メール向け、短め・丁寧の4種類を自動生成
5. **カレンダー登録** — 確定した日程をGoogleカレンダーに直接追加

## セットアップ手順

### 1. Google Cloud Console の設定

#### 1-1. プロジェクトを作成する

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 画面上部の「プロジェクトを選択」→「新しいプロジェクト」
3. プロジェクト名を入力して「作成」

#### 1-2. Google Calendar API を有効化する

1. 左メニュー「APIとサービス」→「ライブラリ」
2. 「Google Calendar API」を検索して選択
3. 「有効にする」をクリック

#### 1-3. OAuth 同意画面を設定する

1. 左メニュー「APIとサービス」→「OAuth 同意画面」
2. User Type: **外部** を選択して「作成」
3. アプリ名・サポートメールを入力（任意の内容でOK）
4. スコープは何も追加せず「保存して次へ」
5. テストユーザーに自分のGmailアドレスを追加（**重要！**）
6. 「保存して次へ」→「ダッシュボードに戻る」

#### 1-4. OAuth クライアントID を作成する

1. 左メニュー「APIとサービス」→「認証情報」
2. 「認証情報を作成」→「OAuth クライアントID」
3. アプリの種類: **ウェブアプリケーション**
4. 名前: 任意（例: 日程調整ツール）
5. 「承認済みのリダイレクトURI」に以下を追加:
   ```
   http://localhost:8501
   ```
6. 「作成」をクリック

#### 1-5. クライアントシークレットをダウンロードする

1. 作成したクライアントIDの右にある「⬇️（ダウンロード）」ボタンをクリック
2. ダウンロードしたJSONファイルを **`client_secrets.json`** という名前に変更
3. このファイルを `schedule-adjustment-tool/` フォルダに配置

> ⚠️ `client_secrets.json` は **絶対にGitHubにpushしないでください**（.gitignoreで除外済み）

---

### 2. .env ファイルの設定

`.env.example` をコピーして `.env` を作成してください。

```bash
cp .env.example .env
```

`.env` を開いて以下を設定します:

```env
# OpenAI APIキー（文章生成に使用。なくてもテンプレートで動作します）
OPENAI_API_KEY=sk-...

# 以下は通常デフォルトのままでOK
GOOGLE_CLIENT_SECRETS_FILE=client_secrets.json
GOOGLE_REDIRECT_URI=http://localhost:8501
```

**OpenAI APIキーの取得方法:**
1. [OpenAI Platform](https://platform.openai.com/api-keys) にアクセス
2. 「Create new secret key」でAPIキーを作成
3. APIキーをコピーして `.env` に貼り付け

> OpenAI APIキーがなくてもアプリは動作します（テンプレートベースの文章が生成されます）

---

### 3. ローカルで起動する

#### 依存パッケージのインストール

```bash
# 仮想環境を作成（推奨）
python -m venv .venv

# 仮想環境を有効化
# Windowsの場合:
.venv\Scripts\activate
# Mac/Linuxの場合:
source .venv/bin/activate

# パッケージをインストール
pip install -r requirements.txt
```

#### アプリを起動

```bash
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動的に開きます。

#### 初回ログイン

1. 「Googleでログイン」ボタンをクリック
2. テストユーザーとして登録したGoogleアカウントでログイン
3. アクセスを許可
4. 自動的にアプリに戻る

## 使い方

1. **ログイン** — Googleアカウントでログイン
2. **条件を入力** — 検索したい期間・時間帯・打ち合わせ時間を選択
3. **検索** — 「空き時間を検索」ボタンをクリック
4. **候補日を確認** — 最大5件の空き時間が表示される
5. **モードを選択** — 「自分から送る」か「相手に返信する」かを選ぶ
6. **文章を生成** — 「送信用文章を生成する」でLINE・メール用の文章が完成
7. **カレンダー登録** — 日程が確定したら候補日を選んでタイトルを入力して登録

## よくある問題

### 「クライアントシークレットファイルが見つかりません」と表示される

`client_secrets.json` を `schedule-adjustment-tool/` フォルダに配置してください。
ファイル名が正確に `client_secrets.json` になっているか確認してください。

### 「このアプリはGoogleで確認されていません」と表示される

開発中のアプリなので正常な表示です。「詳細」→「（アプリ名）に移動」をクリックして続けてください。
OAuth同意画面でテストユーザーに自分のメールアドレスを追加してあれば利用できます。

### 「redirect_uri_mismatch」エラーが出る

Google Cloud ConsoleのリダイレクトURIが `http://localhost:8501` と完全に一致しているか確認してください。
末尾のスラッシュの有無も一致させる必要があります。

### Streamlitが8501以外のポートで起動した場合

ターミナルに表示されているURLを確認して、Google Cloud ConsoleとOAuthのリダイレクトURIを同じポートに合わせてください。
